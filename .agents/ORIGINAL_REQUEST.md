# Original User Request

## 2026-06-11T05:05:34Z

为 Loyea (陪伴型 Android 赛博伴侣) 实现生产级的 MCP 协议客户端，支持多服务器并发管理、工具调用链路打通、智能手表物理感知集成，以及基于后台定时任务的 AI 主动问候推送。

Working directory: D:\CodingProjects\Android\Loyea
Integrity mode: benchmark

## Requirements

### R1. MCP 客户端协议与多服务器管理
- 实现标准的 JSON-RPC over HTTP/SSE 协议客户端，支持同时配置和并发管理多个外部远程 MCP 服务端（如 GitHub MCP、Bilibili MCP 或自定义 Mock MCP 服务）。
- 在设置页中提供优雅的高颜值配置面板，支持服务端地址 of 添加、编辑、删除、以及各服务端连接状态（已连接、连接中、已断开）的实时检测。
- **【Claude美学】** UI 交互与卡片整体风格需模仿 Claude 美学：采用极简精致的莫兰迪色调底盘、轻量高级的磨砂微光边框、平滑弹性的展开/折叠过渡动画，以及温和的点击水波纹回馈。
- 具备断线自动重试机制与断开状态提示，支持展示各个服务器提供的可用 Tools 列表。
- 使用 Android 本地存储（如 SharedPreferences）持久化 MCP 服务端配置。

### R2. LLM 工具调用链路闭环
- 在与大模型对话发送消息时，拉取所有当前已启用 MCP 服务端提供的 Tools 声明，并将它们转换注入到 LLM 请求的 tools 参数中。
- 拦截 LLM 返回 of Tool Calls 响应，通过 JSON-RPC 向对应的 MCP 服务端发起执行请求并获得输出。
- 将工具执行结果作为 `tool` 角色消息追加回上下文队列中，自动发起下一轮 LLM 对话，直至生成最终回答。
- 对接已有的 `ThinkingAndMcpComponents.kt`，在每次执行 Tool Call 时在聊天气泡中实时生成并显示状态为 `RUNNING` 的 `McpCallItem`。执行完毕后，根据结果更新状态为 `SUCCESS` 或 `FAILED`。组件必须支持点击展开查看工具的输入参数与输出结果。

### R3. 智能手表与定位物理感知集成
- 将物理感知选项进行扩展：除“系统时间”外，新增“模拟心率数据”与“实时/模拟定位”参数。
- 定位获取：尝试获取真机定位权限并读取当前位置坐标（经纬度），若无权限或无法获取，则自动回退至预设的模拟坐标（如北京 `39.9042, 116.4074`）或允许用户在设置中自定义输入模拟位置。
- 心率模拟与接口预留：
  - 在物理感知设置中，内置一个“虚拟手表心率生成器”，在开启物理感知时模拟心率数据波动（基准 `60-100 bpm`），支持通过设置页开关切换“静息”或“运动”模拟状态。
  - **【硬件预留】** 提前预留好对接真实智能手表的 UI 交互开关（如“手表同步”切换项）以及底层数据通信的程序接口抽象（如 `WatchDataRepository` 接口类与其 Mock 实现），以便后期直接接入 WearOS/Health Connect SDK。
- 数据注入：在物理感知开启时，每次向 LLM 发送请求前，将当前的系统时间、当前心率、运动状态、当前地理位置以结构化段落 `[Physical Context]` 的形式，动态拼接并注入到系统 Prompt 中（可集成到 `PromptAssembler.kt`）。

### R4. 后台定时主动问候推送
- 使用 `WorkManager`（或配合 `AlarmManager` 实现精确闹钟定时）在后台注册每日定时任务（例如睡前 22:30 或特定纪念日/生日）。
- 后台任务唤醒时，在后台静默发起大模型请求。Prompt 拼接模板中加入环境与物理感知上下文（如当前是深夜、模拟心率表现等），命令 AI 作为 Loyea 角色生成一段 50 字以内的主动温暖关怀或问候。
- AI 生成回复后，Loyea 客户端通过系统状态栏发送 Android 消息通知（Notification），通知样式与动作细节也需贯彻 Claude 式优雅。
- 用户点击通知后，自动打开 App 聊天页，且该条后台问候消息已被以 AI 消息的形式插入到当前的活跃会话历史中并完成持久化。

## Acceptance Criteria

### MCP 服务端管理与状态 (R1)
- [ ] 可添加并启用多个标准的 SSE MCP 地址（例如：`http://10.0.2.2:8000/sse`）。
- [ ] UI 设置列表中显示各个服务器的状态（在线、离线），且点击某服务器能查看其暴露的所有 Tools 名称与描述。
- [ ] 当服务器掉线时，UI 连接状态能迅速流转为“离线”；当重新上线时可手动触发重连或自动重试成功。
- [ ] UI 卡片与动作呈现高度拟真的 Claude 极简高档感。

### 工具调用链路与 UI 联动 (R2)
- [ ] 对话时如果大模型决定调用工具，UI 聊天列表能生成并渲染对应的 `McpCallItem`；在执行期间，齿轮保持匀速旋转（`RUNNING`）；执行成功后图标转为绿勾（`SUCCESS`），失败为红感叹号（`FAILED`）。
- [ ] 工具调用卡片支持点击手势，展开后能完整、清晰地展示该工具的 Input（JSON参数）和 Output（返回结果）。
- [ ] 大模型最终能结合工具的输出，在打字机效果中输出正确的最终回答。

### 物理感知数据获取与注入 (R3)
- [ ] 开启物理感知后，LLM 请求 of system prompt 中必须成功包含类似下面的段落：
  ```
  [Physical Context]
  Current Time: 2026-06-11 23:00:00
  Heart Rate: 72 bpm (Resting)
  Location: 39.9042, 116.4074
  ```
- [ ] 修改物理感知中的“静息/运动”状态或在有权限下改变位置，发送新消息时 LLM 接收到的 payload 中心率和坐标值应随之更新。
- [ ] 代码中暴露出独立的 `WatchDataRepository` 接口，并有明确的 Mock 注入点。

### 后台定时主动问候与通知拉起 (R4)
- [ ] 在开发者调试选项中或通过 `adb shell cmd jobscheduler run` 手动触发问候 WorkManager 任务。
- [ ] 任务触发后，系统状态栏成功弹出 Loyea 的通知（标题为角色名，内容为 AI 对应生成的问候语）。
- [ ] 点击通知能拉起 Loyea `MainActivity`，聊天界面新增了此条 AI 问候消息，且可以正常存入本地 JSON 缓存。
